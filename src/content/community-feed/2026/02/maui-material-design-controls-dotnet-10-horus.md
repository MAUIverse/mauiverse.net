Agustin Bonilla from Horus Software introduces their open-source Material Design control library for .NET MAUI — originally born as a Xamarin plugin, now rebuilt for .NET MAUI with 89K downloads. The post makes a compelling case for why a shared UI foundation matters more as your app scales.

## What you'll learn

- **Dramatic code reduction** — a side-by-side comparison shows 50 lines of raw XAML reduced to 21 lines with the library for the same login form
- How the library evolved from their Xamarin `Plugin.MaterialDesignControls` into a modern .NET MAUI architecture
- Why .NET 10 support matters practically — long-term support, better performance, and fewer unexpected issues during upgrades
- Built-in components like `MaterialTextField`, `MaterialButton`, and `MaterialSnackbar` with binding-friendly APIs
- How consistent controls reduce visual bugs and repetitive UI code as projects grow from first release to long-term maintenance
- The library is open-source, production-tested in real client apps at Horus, and actively maintained on [GitHub](https://github.com/HorusSoftwareUY/Maui.MaterialDesignControls)
