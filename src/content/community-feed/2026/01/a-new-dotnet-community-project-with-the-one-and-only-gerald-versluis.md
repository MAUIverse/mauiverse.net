Starting a community project is exciting, but the first architecture and governance choices determine whether people can actually contribute. This conversation is a candid planning session on building something useful, sustainable, and welcoming for the .NET ecosystem.

### What you will learn

- How to define an initial problem statement without over-scoping the first release
- What contributor onboarding needs early to avoid bottlenecks later
- How maintainers think about project structure, ownership, and roadmap trade-offs
- Why clear boundaries and documentation matter before writing lots of code
